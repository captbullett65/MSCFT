BRAINSTORM_TEMPLATE_V1

OBJECTIVE:
Define what you’re trying to figure out in one sentence.

SCOPE:
Define boundaries (timeframe, domain, constraints) in one short line.

LENGTH RULE:
Target up to 1,200 words total.
No double spaces.
No extra blank lines.
Single line breaks only.

STRUCTURE:

1. Core Question
   State the exact question or problem in one sentence.

2. Known Facts
   List 3–7 short lines of what is known or assumed.
   No explanations, just statements.

3. Unknowns / Gaps
   List 3–7 short lines of what is unclear or missing.

4. Hypotheses
   List 2–5 possible explanations or outcomes.
   Each one must be one sentence only.

5. Drivers / Variables
   List key factors influencing the outcome (3–6 items).
   Keep each to one line.

6. Constraints
   List hard limits (time, resources, rules, external conditions).

7. Quick Scenarios
   Write 2–3 short scenario lines:
   Best case:
   Base case:
   Worst case:

8. Preliminary Takeaway
   Write 1–2 sentences summarizing your current leaning.

STYLE RULES:
No inline citations, no references, no links, no brackets, no footnotes of any kind.
No double spacing.
No filler language.
No repeated ideas.
One idea per line.
Keep sentences short and declarative.
Do not explain everything—capture signal, not noise.

OUTPUT FORMAT:
Core Question: [text]
Known Facts: [line-separated]
Unknowns: [line-separated]
Hypotheses: [line-separated]
Drivers: [line-separated]
Constraints: [line-separated]
Scenarios: [3 short lines]
Takeaway: [1–2 sentences]

CUT PASS:
Reduce total word count by approximately 20% without removing meaning.
Eliminate filler, redundancy, and low-value modifiers only.

FINALIZATION REQUIREMENT:
Remove empty lines
Enforce single spacing
Remove citations/links/references/footnotes/brackets
Remove template headers and metadata from generated output
Apply CUT PASS
Verify structure integrity
Verify section order
Output only the OUTPUT FORMAT section
